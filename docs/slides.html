🐛 Spot the Bug

What does this print?

int marks = 450;
int total = 500;
System.out.println("Percentage: " + marks/total * 100);
  • A) 90
  • B) 90.0
  • C) 0
  • D) Compilation Error

💥 Integer division! 450/500 = 0, then 0 * 100 = 0.
Fix: (double) marks / total * 100

OOP — Class, Inheritance & Subclass

🏗 Use a Class when

Multiple fields + behavior belong together
Employee — name, salary, computePay()

🧬 Use Inheritance when

"IS-A" relationship exists
SavingsAccount IS-A Account

🔧 Use Subclass when

Behavior differs by type
FDAccount lock-in vs CurrentAccount overdraft

IS-A → Inheritance
Dog IS-A Animal  ·  SavingsAccount IS-A Account
HAS-A → Composition
Car HAS-A Engine  ·  Employee HAS-A Address

Spring Boot — Client-Server Architecture

⚙️ Service — ALL business logic lives here
↕
🗄 Repository — only fetches & saves
↕ SQL (JPA)
🐬 MySQL Database — persists data

Spring Boot — Code Skeleton

Controller (thin — no logic!)

@RestController
@RequestMapping("/api/submissions")
public class SubmissionController {
  @Autowired SubmissionService svc;

  @PostMapping
  public ResponseEntity create(
      @RequestBody Submission s) {
    return ResponseEntity.ok(
        svc.create(s));
  }
}

Service (logic owner)

@Service
public class SubmissionService {
  @Autowired SubmissionRepository repo;

  public Submission create(Submission s) {
    if (s.getStudentName() == null)
      throw new IllegalArgumentException(
          "Name required");
    s.setStatus("SUBMITTED");
    return repo.save(s);
  }
}

🏗 Quiz — Which Layer? Interactive

Q1: Your code checks "if score > 40, student passes". Where does this go?

  • A) Controller — it's important!
  • B) Service — business logic lives here
  • C) Repository — save it to DB
  • D) main() method 😅

✅ Service layer — any business rule goes in Service. Controller just passes it through.


Q2: A waiter in a restaurant is like which Spring Boot layer? 🍽

  • A) Controller — takes order, delivers food
  • B) Service — the chef cooking
  • C) Repository — the fridge storing ingredients
  • D) MySQL — the pantry

✅ Controller = waiter (takes request, delivers response). Service = chef (does the real work). Repository = fridge (stores stuff). DB = pantry 😄

Integration Blueprint

✅ Happy Path

1. Request arrives at Controller
2. Controller validates required fields
3. Service checks business rules
4. Repository saves to MySQL
5. Return 200 OK + data

❌ Error Paths

Missing field → 400 Bad Request
Rule violated → 422 + message
Not found → 404
DB error → 500

⚡ Design happy path AND failure path together.
